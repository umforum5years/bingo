# Umforum Bingo

Приложение для создания и проведения музыкальных бинго-игр.

## Структура приложения

Приложение состоит из трёх страниц:

1. **Конструктор** (`/design`) — создание бланков для бинго
2. **Предпросмотр** (`/print-preview`) — просмотр, настройка и печать бланков
3. **Игра** (`/play`) — проведение игры с готовыми бланками

---

## Страница "Конструктор"

### 1. Загрузка музыкальных файлов

**Кнопка:** `Выбрать треки` (иконка 📁)

- Выберите файлы в формате `.mp3` с вашего устройства
- **Важно:** имя исполнителя извлекается из названия файла **до первого дефиса**

  Пример: `The Beatles - Hey Jude.mp3` → исполнитель: `The Beatles`

- **Внимание:** если в названии исполнителя есть дефис (например, `Blink-182`), переименуйте файл, заменив дефис на другой символ (например, `Blink_182 - Song.mp3`)

### 2. Нумерация исполнителей

**Кнопка:** `Назначить номера` (иконка 🔢)

- Присваивает случайные номера всем найденным исполнителям
- Номера отображаются в таблице рядом с именами исполнителей

**Таблица исполнителей:**
- Содержит колонки: `№`, `Исполнитель`
- Можно фильтровать через поле поиска
- Можно редактировать номера вручную

**Сохранение и загрузка списка исполнителей:**

- **Кнопка:** `Сохранить список (json)` (иконка 💾) — сохраняет список в JSON для повторного использования
- **Кнопка:** `Загрузить список (json)` (иконка 📤) — загружает ранее сохранённый список исполнителей из JSON
- **Кнопка:** `Загрузить список (txt)` (иконка 📄) — загружает список исполнителей из обычного текстового файла (каждый исполнитель на новой строке)
- **Кнопка:** `Скачать список (txt)` (иконка 📥) — скачивает текущий список исполнителей в виде текстового файла с нумерацией

### 3. Генерация бланков

**Параметры перед генерацией:**

- **Размер сетки:** выпадающий список (5×5, 6×6, ..., 10×10)
- **Свой размер:** чекбокс + поле ввода для нестандартного размера (минимум 2, максимум 20)
- **Отображение:** выбор формата ячеек:
  - `Название и номер` — показывает и то, и другое
  - `Только название` — только имя исполнителя
  - `Только номер` — только номер
- **Количество бланков:** числовое поле (сколько бланков создать)

**Кнопка:** `Создать бланки` (иконка ➕)

- Генерирует указанное количество уникальных бланков
- Каждый бланок содержит случайную выборку исполнителей

### 4. Сохранение бланков

После генерации становятся доступны кнопки:

**Кнопка:** `Сохранить в JSON` (иконка 💾)

- Сохраняет бланки в формате JSON для использования в режиме "Игра" или "Предпросмотр"
- Файл содержит: номер бланка, размер сетки, режим отображения, номера и имена исполнителей

**Кнопка:** `Загрузить из JSON` (иконка 📤)

- Загружает ранее сохранённые бланки

**Кнопка:** `Очистить` (иконка 🗑️)

- Удаляет все сгенерированные бланки

## Страница "Предпросмотр"

Страница для просмотра, настройки и печати сгенерированных бланков.

### 1. Загрузка бланков

**Кнопка:** `Загрузить бланки` (иконка 📤)

- Загружает JSON-файл с бланками, созданный на странице "Конструктор"

### 2. Настройка отображения

После загрузки бланков становятся доступны элементы управления:

**Ориентация:**
- **Портретная (A4)** — вертикальная ориентация страницы
- **Альбомная (A4)** — горизонтальная ориентация страницы

**Отображение:**
- `Название и номер` — показывает и то, и другое
- `Только название` — только имя исполнителя
- `Только номер` — только номер

**Размер шрифта номера:**
- Поле ввода для настройки размера шрифта номера в ячейке (от 4 до 72 pt)
- Значение по умолчанию: 14 pt

**Размер шрифта названия:**
- Поле ввода для настройки размера шрифта названия исполнителя (от 4 до 72 pt)
- Значение по умолчанию: 8 pt

### 3. Фон бланков

**Кнопка:** `Загрузить фон` (иконка 🖼️)

- Загружает изображение для фона бланков
- Поддерживаются любые форматы изображений

**Кнопка:** `Очистить фон` (иконка 🗑️)

- Удаляет установленный фон

### 4. Экспорт и печать

**Кнопка:** `Сохранить как PDF` (иконка 📄)

- Сохраняет все бланки в один PDF-файл
- Каждый бланк на отдельной странице A4
- Ориентация страницы соответствует выбранной (портретная/альбомная)
- Высокое качество изображения (300 DPI)

**Кнопка:** `Сохранить как Word` (иконка 📝)

- Экспортирует бланки в файл `.docx`
- Размер шрифта соответствует значениям, заданным в контролах
- Каждый бланк на отдельной странице
- Ориентация страницы соответствует выбранной

**Кнопка:** `Сохранить как JPG` (иконка 🖼️)

- Сохраняет каждый бланк как отдельное JPG-изображение
- Высокое качество (95% сжатие)
- Изображения скачиваются по одному с небольшой задержкой
- Подходит для отправки в мессенджерах или соцсетях

**Кнопки сохранения текущего бланка** (отображаются в навигационной панели):

- **`Сохранить бланк в PDF`** — сохраняет только текущий отображаемый бланк в PDF
- **`Сохранить бланк в JPG`** — сохраняет только текущий отображаемый бланк в JPG
- Удобно для быстрой печати или отправки одного бланка

**Навигация по бланкам:**

- Кнопки `Первая`, `Предыдущая`, `Следующая`, `Последняя` — переключение между бланками
- Индикатор показывает текущий бланк и общее количество
- При сохранении всех бланков отображается прогресс выполнения
- Кнопка `Отменить` позволяет прервать сохранение (уже созданные файлы сохраняются)

## Страница "Игра"

### 1. Загрузка данных

**Кнопка:** `Загрузить список исполнителей` (иконка 👥)

- Загружает JSON-файл со списком исполнителей и номерами
- Создаётся на странице "Конструктор" через кнопку "Сохранить"

**Кнопка:** `Загрузить список бланков` (иконка 📋)

- Загружает JSON-файл с бланками
- Создаётся на странице "Конструктор" через кнопку "Сохранить в JSON"

### 2. Сохранение и загрузка игры

**Кнопка:** `Сохранить игру` (иконка 💾)

- Сохраняет текущее состояние игры в JSON-файл
- Файл содержит: список исполнителей, бланки, выпавшие номера, достижения, настройки
- Позволяет продолжить игру позже или на другом компьютере

**Кнопка:** `Загрузить игру` (иконка 📤)

- Загружает ранее сохранённое состояние игры из JSON-файла
- Восстанавливает все параметры: выпавшие номера, прогресс бланков, достижения

**Автосохранение:**

- Состояние игры автоматически сохраняется в localStorage браузера
- При перезагрузке страницы игра восстанавливается автоматически
- При загрузке новых данных (исполнителей или бланков) автосохранение сбрасывается

### 3. Розыгрыш номеров

**Кнопка:** `Получить номер` (иконка 🎫)

- Выдаёт случайный номер из списка исполнителей, который ещё не выпадал
- Добавляет номер в список выпавших. При наведении на номер можно увидеть имя исполнителя.

**Поле ручного ввода:**

- Поле `Номер` + кнопка `Добавить`
- Для ручного ввода номера исполнителя

**Кнопка:** `Сбросить игру` (иконка 🔄)

- Очищает список выпавших номеров (начинает игру заново)

### 4. Настройка подсчёта линий

**Чекбокс:** `Учитывать диагонали при подсчёте линий`

- По умолчанию включено
- Если снято — диагонали не учитываются при подсчёте заполненных линий для достижений
- Настройка сохраняется в localStorage и восстанавливается при загрузке игры

### 5. Отображение результатов

**Блок "Выпавшие номера":**

- Показывает все выпавшие номера в виде кружков
- При наведении на номер появляется подсказка с именем исполнителя
- **Клик по номеру** — удаляет номер из выпавших (с подтверждением) и возвращает его в общий набор для повторного розыгрыша
- При удалении номера список карточек автоматически обновляется

**Блок "Подходящие бланки":**

- Отображает бланки, в которых есть совпадения с выпавшими номерами
- **Сортировка:** переключатель между двумя режимами:
  - `По количеству совпадений` — сортирует по убыванию количества совпавших номеров
  - `По заполненным линиям` — сортирует сначала по сумме заполненных строк/столбцов/диагоналей, при равенстве — по количеству совпадений

**Карточка бланка содержит:**

- Номер бланка и размер сетки (например, `№1 5×5`)
- Процент совпадения (например, `60%`)
- Прогресс-бар заполненности
- Количество совпавших номеров (например, `15 из 25`)
- Сетка с номерами (выпавшие подсвечены цветом)
- Имена исполнителей, которые совпали
- **Индикаторы заполненных линий:**
  - ↔️ — заполненные строки (например, `2/5`)
  - ↕️ — заполненные столбцы (например, `1/5`)
  - 🔀 — заполненные диагонали (всего 2: главная и побочная)
  - Подсвечиваются зелёным, если есть хотя бы одна заполненная линия
- **Индикатор нового заполнения:** 🔔 — появляется рядом с номером бланка, если после последнего выпавшего номера увеличилось количество заполненных линий (исчезает после следующего розыгрыша)

### 6. Достижения бланков

**Иконка кубка 🏆** рядом с заголовком "Подходящие бланки":

- Показывает количество достижений
- Подсвечивается золотым градиентом при наличии достижений
- Клик открывает модальное окно со списком всех достижений

**Виды достижений:**

- **1 линия** — "Бланк №X собрал первую линию! 🎉"
- **2 линии** — "Бланк №X собрал две линии! 🎉🎉"
- **3 линии** — "Бланк №X собрал три линии! 🎉🎉🎉"
- **4 линии** — "Бланк №X собрал четыре линии! 🎉🎉🎉🎉"
- **5 линий** — "Бланк №X собрал пять линий! 🔥🔥🔥"
- **Все числа (БИНГО)** — "Бланк №X собрал все числа! 🎊🎊🎊"

**Особенности:**

- Уведомление показывается только для первого бланка, достигшего milestone
- Если несколько бланков одновременно достигли milestone — показываются все
- Повторные достижения того же уровня не показываются
- Достижения сохраняются в localStorage и в файле сохранения игры

**Всплывающие уведомления:**

- При достижении milestone показывается toast-уведомление в правом верхнем углу
- Уведомление исчезает через 5 секунд

## Типовой сценарий использования

### Подготовка (страница "Конструктор")

1. Нажмите `Выбрать треки`, выберите MP3-файлы
2. Нажмите `Пронумеровать` для присвоения номеров
3. (Опционально) Нажмите `Сохранить` для сохранения списка исполнителей
4. Настройте параметры бланков (размер, количество, отображение)
5. Нажмите `Создать бланки`
6. Нажмите `Сохранить в JSON` для использования в игре

### Печать бланков (страница "Предпросмотр")

1. Нажмите `Загрузить бланки`, выберите JSON-файл с бланками
2. Настройте ориентацию страницы (портретная/альбомная)
3. Выберите режим отображения (название/номер/оба)
4. Настройте размер шрифта для номера и названия исполнителя
5. (Опционально) Загрузите фоновое изображение
6. Используйте кнопки экспорта:
   - `Сохранить как PDF` — для создания PDF-файла со всеми бланками
   - `Сохранить как JPG` — для получения изображений всех бланков
   - `Сохранить бланк в PDF/JPG` — для быстрого сохранения текущего бланка

### Проведение игры (страница "Игра")

1. Нажмите `Загрузить список исполнителей`, выберите JSON-файл
2. Нажмите `Загрузить список бланков`, выберите JSON-файл с бланками
3. (Опционально) Настройте учёт диагоналей для достижений
4. Нажимайте `Получить номер` или вводите номера вручную
5. Следите за заполненностью бланков в блоке "Подходящие бланки"
6. (Опционально) Нажмите `Сохранить игру` для сохранения прогресса
7. Игра продолжается до полного заполнения строки/столбца у одного из игроков
8. При достижении БИНГО срабатывает салют 🎆

### Продолжение игры после перерыва

**Вариант 1: Автосохранение**

- Просто откройте страницу игры — состояние восстановится автоматически из localStorage

**Вариант 2: Загрузка из файла**

1. Нажмите `Загрузить игру`
2. Выберите ранее сохранённый JSON-файл
3. Игра восстановится в том же состоянии, в котором была сохранена

**Вариант 3: Перенос на другой компьютер**

1. На старом компьютере: нажмите `Сохранить игру`, скачайте файл
2. Перенесите файл на новый компьютер (USB, облако, почта)
3. На новом компьютере: нажмите `Загрузить игру`, выберите файл
4. Игра продолжится с того же места

## Технические требования

- Браузер: Chrome, Firefox, Safari, Edge (последние версии)

## PWA (Progressive Web App)

Приложение поддерживает технологию PWA и может работать без подключения к интернету.

### Установка приложения

**На компьютере:**
- В браузере Chrome/Edge нажмите на значок установки в адресной строке
- Или выберите меню браузера → «Установить Umforum Bingo»

**На телефоне (Android):**
- Откройте меню браузера → «Установить приложение» или «Добавить на главный экран»

**На iPhone (iOS):**
- Нажмите кнопку «Поделиться» → «На экран "Домой"»

### Автономная работа

- После первого посещения приложение кешируется и становится доступным без интернета
- При запуске без соединения данные загружаются из кеша
- Все созданные бланки и списки исполнителей сохраняются локально

### Автоматическое обновление

- При наличии новой версии приложения в правом нижнем углу появится уведомление
- Нажмите кнопку **«Обновить»** для применения обновлений
- Страница автоматически перезагрузится с новой версией

### Что кешируется

- **Приложение:** JavaScript, CSS, HTML файлы
- **Ресурсы:** Иконки, изображения, шрифты
- **Данные:** JSON файлы с исполнителями и бланками
